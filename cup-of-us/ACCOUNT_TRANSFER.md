# Moving Cup of Us to another ChatGPT account

## In the current account

1. Keep this repository private.
2. Confirm `npm test` passes.
3. Push the latest `main` branch.
4. Rotate any secret that has appeared in chat or screenshots.

## In the new account

1. Install and connect the GitHub plugin.
2. Grant it access to this private repository.
3. Open the repository in Codex.
4. Ask Codex to install dependencies, read this file, and run the tests locally.
5. Create a new Sites project owned by the new account.
6. Replace the `project_id` in `.openai/hosting.json` with the new opaque project ID generated by Sites. Do not invent one.
7. Add fresh environment variables to the new hosting project.
8. Publish only after the local build, tests, and responsive checks pass.

## Data transfer

Source code migrations do not move production records automatically. Export and import D1 data separately if saved cups, profiles, community posts, or analytics must be preserved.

Do not commit database exports when they contain user data.

## Prompt for the new account

```text
Open the Cup of Us repository. Read README.md and ACCOUNT_TRANSFER.md completely. Install dependencies, create .env.local from .env.example without inventing credentials, run the build and tests, and inspect the responsive UI locally. Create a new Sites project owned by this account and update only the new project binding. Do not publish until every test passes. Never commit secrets.
```
